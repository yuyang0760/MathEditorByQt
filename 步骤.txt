1. 插入位置始终在文档开头
原因
鼠标点击 DocumentView 改变选区后，DocumentView 发出 selectionChanged 信号，但 TextEditorWidget::onSelectionChanged 槽函数中没有将新的选区同步给 SelectionController。导致 InputController 通过 m_selectionController->selection() 获取的始终是初始值 (0,0,0)，因此所有插入操作都在第一个段落的开头进行。

修复
在 onSelectionChanged 中添加对 SelectionController::setSelection 的调用：

cpp
void TextEditorWidget::onSelectionChanged(const Selection &selection) {
    // 同步到选区控制器
    if (m_selectionController) {
        m_selectionController->setSelection(selection);
    }

    m_documentView->ensureCursorVisible();
    updateStatusBar();
}
SelectionController::setSelection 内部会进行相等性判断，不会引起死循环。
